# Checklist de Acessibilidade (a11y / WCAG)

Referência do passo 5 de `site-organizacao`. Acessibilidade desde a fundação — não é polimento final.
Alvo: WCAG 2.1 AA. Boa parte é verificável; o que for visual exige conferência manual.

## Estrutura semântica
- [ ] Landmarks: `<header>`, `<nav>`, `<main>` (um por página), `<section>`, `<article>`, `<footer>`.
- [ ] Hierarquia de headings correta (`<h1>` único → `<h2>` → `<h3>`, sem pular níveis).
- [ ] `<html lang="pt-BR">` definido.

## Imagens e mídia
- [ ] Toda `<img>` com `alt` descritivo (ou `alt=""` se decorativa).
- [ ] Ícones interativos com rótulo acessível (`aria-label`).

## Interação e teclado
- [ ] Tudo operável por teclado (Tab/Shift+Tab/Enter/Esc); ordem de foco lógica.
- [ ] **Foco visível** em todos os elementos focáveis (não remover `outline` sem substituto).
- [ ] Abas (tabs) com `role="tablist"`/`tab`/`tabpanel` e navegação por setas.
- [ ] Botão é `<button>`, link é `<a href>` — não `<div onclick>`.

## Cor e contraste
- [ ] Contraste texto/fundo ≥ **4.5:1** (≥ 3:1 para texto grande).
- [ ] Informação não transmitida **só** por cor (ex.: erro com ícone + texto).

## Formulários
- [ ] Todo input com `<label>` associado (`for`/`id`).
- [ ] Erros anunciados (texto + `aria-describedby`/`aria-live`).

> Ferramentas: axe DevTools / Lighthouse (aba Accessibility) para o varrimento; teste manual de teclado para o foco.
